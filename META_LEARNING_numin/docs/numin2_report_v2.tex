\section{Introduction \& Problem Setup}
%==============================================================================

\subsection{The Numin2 Benchmark}
We address the problem of predicting relative stock rankings for the \textbf{Nifty-50 index} (50 largest Indian stocks by market capitalization) using daily return data. This is formulated as a meta-learning problem where each \emph{month} of trading constitutes a separate task with potentially different market dynamics.

\begin{tcolorbox}[colback=blue!3,colframe=blue!40!black,title=\textbf{Task Definition}]
\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Input}: Sliding window of $W\!=\!50$ consecutive trading days across 50 stocks: $X \in \mathbb{R}^{50 \times 50}$
  \item \textbf{Target}: Rank ordering of 50 stocks by next-day returns: $y \in \{0, 1, \ldots, 49\}^{50}$
  \item \textbf{Support set}: First $K\!=\!5$ days of the month (for task adaptation)
  \item \textbf{Query set}: Remaining days of the month (for evaluation)
  \item \textbf{Metric}: Spearman rank correlation $\rho_s$ between predicted and actual rankings
  \item \textbf{Data split}: Chronological 70/15/15 train/val/test (no temporal leakage)
  \item \textbf{Total tasks}: 22 monthly tasks from the parquet dataset
\end{itemize}
\end{tcolorbox}

\subsection{Why Meta-Learning for Finance?}
Financial markets exhibit \textbf{non-stationarity} (Lecture~1: $p(y|x)$ changes over time) and \textbf{concept drift} --- the statistical properties of returns shift between months due to changing market regimes, policy events, and sector rotations. Traditional supervised learning assumes i.i.d.\ data and fails when the distribution shifts.

Meta-learning directly addresses this by learning \textbf{how to adapt}: rather than memorizing patterns from historical data, the model learns an initialization $\theta$ that can be quickly fine-tuned to new market conditions using just a few days of support data. This implements Lecture~10's \emph{Financial Meta-Learning Setup}: 50 assets, feature vectors, strategy positions~\cite{numin2024}.

\subsection{Formulation (Lecture~5)}
Following the formal meta-learning framework:
\begin{equation}
  \theta^* = \arg\min_\theta \sum_{k=1}^{K} \mathcal{L}\bigl(f_{\theta'_k}, \mathcal{D}_{\text{query}}^k\bigr) \quad \text{where} \quad \theta'_k = \text{Adapt}(\theta, \mathcal{D}_{\text{support}}^k)
\end{equation}
Lecture~5: \emph{``If the goal $G$ is to learn from a small number of examples, then train for that goal.''}

  \item \textbf{LSTM+Transformer ensemble with learned weights}: A Reptile-trained ensemble combining two different encoder architectures with softmax-weighted combination --- a practical hybrid approach.
\end{enumerate}
\end{tcolorbox}

\section{Model Architecture}
%==============================================================================

All methods share a common \textbf{encoder--decoder} backbone, with the meta-learning algorithm determining how adaptation occurs.

\subsection{Encoder: Bidirectional LSTM with Attention}
The encoder processes the $50 \times 50$ input (window $\times$ stocks) through:
\begin{enumerate}[nosep]
  \item \textbf{Input projection}: $\text{Linear}(50 \to H)$ maps each day's 50-stock vector to hidden dimension $H$.
  \item \textbf{Bidirectional LSTM}: 2-layer BiLSTM captures temporal dependencies in both directions:
  \begin{equation}
    \vec{h}_t = \text{LSTM}_{\rightarrow}(x_t, \vec{h}_{t-1}), \quad \overleftarrow{h}_t = \text{LSTM}_{\leftarrow}(x_t, \overleftarrow{h}_{t+1}), \quad h_t = [\vec{h}_t; \overleftarrow{h}_t]
  \end{equation}
  \item \textbf{Multi-head self-attention}: 4-head attention over temporal positions captures long-range dependencies:
  \begin{equation}
    \text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V, \quad \text{with } Q=K=V=H_{\text{LSTM}}
  \end{equation}
  \item \textbf{LayerNorm + residual}: $\text{out} = \text{LN}(H_{\text{LSTM}} + H_{\text{Attn}})$
  \item \textbf{Global average pooling}: $\bar{h} = \frac{1}{W}\sum_{t=1}^{W} h_t \in \mathbb{R}^{2H}$
\end{enumerate}

\subsection{Decoder: Rank Prediction Head}
A 3-layer MLP with dropout and LayerNorm:
\begin{equation}
  \text{Decoder}: \mathbb{R}^{2H} \xrightarrow{\text{Linear+ReLU+Drop+LN}} \mathbb{R}^{H} \xrightarrow{\text{Linear+ReLU+Drop+LN}} \mathbb{R}^{H} \xrightarrow{\text{Linear}} \mathbb{R}^{50 \times 50}
\end{equation}
The output is reshaped to $(B, 50, 50)$ --- for each of 50 stocks, a probability distribution over 50 rank positions. Loss: \textbf{cross-entropy} per stock, vectorized as \texttt{F.cross\_entropy(logits.reshape(B*S, C), targets.reshape(B*S))}.

\subsection{Correlation Metric: Soft Expected Rank}
Instead of using \texttt{argmax} (which can produce duplicate ranks), we compute the \textbf{expected rank}:
\begin{equation}
  \hat{r}_i = \sum_{j=0}^{49} j \cdot \text{softmax}(\text{logits}_i)_j
\end{equation}
This produces continuous rank predictions suitable for Spearman correlation without ties.

\section{Methods Implemented}
%==============================================================================

We implemented \textbf{11 methods} spanning all three meta-learning families from Lecture~10's taxonomy, plus ensemble approaches.

%----------------------------------------------------------------------
\subsection{MAML --- Model-Agnostic Meta-Learning~\cite{finn2017}}
%----------------------------------------------------------------------

\textbf{Family}: Optimization-based (Lecture~6). \textbf{File}: \texttt{numin\_maml.py}.

\textbf{Algorithm}: Learn an initialization $\theta$ such that a few gradient steps produce good task performance:
\begin{equation}
  \theta'_k = \theta - \alpha \nabla_\theta \mathcal{L}(\mathcal{D}_{\text{support}}^k; \theta), \qquad \theta \leftarrow \theta - \beta \nabla_\theta \mathcal{L}(\mathcal{D}_{\text{query}}^k; \theta'_k)
\end{equation}

\textbf{Implementation}: We use \texttt{torch.func.functional\_call} with \texttt{create\_graph=True} for proper second-order gradient computation. cuDNN is disabled during the inner loop (\texttt{torch.backends.cudnn.flags(enabled=False)}) because LSTM does not support double backward with cuDNN kernels. Efficient SDPA is also disabled for attention layers.

\textbf{Hyperparameters}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=3, hidden\_dim=128, epochs=50.

\textbf{Result}: Val=0.757, \textbf{Test=0.565}. MAML works but is outperformed by first-order methods, likely due to gradient noise amplification through second-order derivatives on noisy financial data.

%----------------------------------------------------------------------
\subsection{FOMAML --- First-Order MAML}
%----------------------------------------------------------------------

\textbf{Family}: Optimization-based (Lecture~6). \textbf{File}: \texttt{numin\_fomaml.py}.

\textbf{Algorithm}: Drops second-order gradients for computational efficiency:
\begin{equation}
  \theta \leftarrow \theta - \beta \nabla_{\theta'_k} \mathcal{L}(\mathcal{D}_{\text{query}}^k; \theta'_k) \quad \text{(stop gradient through inner loop)}
\end{equation}

\textbf{Implementation}: Uses \texttt{deepcopy(model)} to create a task-specific copy. After inner-loop adaptation via SGD, the first-order gradients from the query loss on the adapted model are copied back to the meta-model parameters. This avoids computing Hessians entirely.

\textbf{Hyperparameters}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=5, hidden\_dim=256, epochs=100.

\textbf{Result}: Val=0.645, \textbf{Test=0.631}. Significantly better than full MAML --- the noise amplification from second-order gradients is avoided, and the simpler update provides more stable learning on financial data.

%----------------------------------------------------------------------
\subsection{ANIL --- Almost No Inner Loop~\cite{raghu2020}}
%----------------------------------------------------------------------

\textbf{Family}: Optimization-based. \textbf{File}: \texttt{numin\_anil.py}.

\textbf{Key insight} (Raghu et al.~\cite{raghu2020}): MAML's success comes from \emph{feature reuse}, not rapid body adaptation. Only the \textbf{head} (last linear layer) needs task-specific adaptation:
\begin{align}
  \text{Inner loop}: \quad & \text{head}'_k = \text{head}_k - \alpha \nabla_{\text{head}} \mathcal{L}_{\text{support}} \quad \text{(body frozen)} \\
  \text{Outer loop}: \quad & [\text{body}; \text{head}] \leftarrow [\text{body}; \text{head}] - \beta \nabla_{[\text{body};\text{head}]} \mathcal{L}_{\text{query}}(\theta'_k) \quad \text{(all updated)}
\end{align}

\textbf{Implementation}: Uses \texttt{functional\_call} with \texttt{create\_graph=True}. The inner loop only computes gradients for parameters containing ``decoder'' in their key name. The query loss backpropagates through the inner loop to update \emph{all} parameters (body + head) via meta-gradients.

\textbf{Architecture}: Body = BiLSTM + MultiHeadAttention + LayerNorm + projection. Head = \texttt{nn.Linear(hidden, 50*50)}.

\textbf{Hyperparameters}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=3, hidden\_dim=256, epochs=50.

\textbf{Result}: Val=0.758, \textcolor{bestgreen}{\textbf{Test=0.670}}. \textbf{Best method overall.} ANIL confirms the feature reuse hypothesis for financial data: the LSTM body learns market-regime-agnostic temporal features, and only the ranking head needs task-specific adaptation.

%----------------------------------------------------------------------
\subsection{Reptile~\cite{nichol2018}}
%----------------------------------------------------------------------

\textbf{Family}: Optimization-based (Lecture~8). \textbf{File}: \texttt{numin\_reptile.py}.

\textbf{Algorithm}: Run SGD on a task, then interpolate toward the adapted weights:
\begin{equation}
  \boxed{\theta \leftarrow \theta + \epsilon(\tilde{\theta} - \theta)} \quad \text{where } \tilde{\theta} = \text{SGD}^K(\theta, \mathcal{D}_{\text{support}})
\end{equation}

Lecture~8: \emph{``Drop all second-order terms. Move the initialization $\theta$ toward the adapted parameters.''}. Geometric intuition: $\theta$ moves toward the \textbf{centroid} of all task-specific optima.

\textbf{Key feature}: Linear $\epsilon$ decay: $\epsilon_t = \epsilon_0 \cdot (1 - t/T)$ following the original paper's recommendation.

\textbf{Hyperparameters}: inner\_lr=0.01, outer\_lr=0.1, inner\_steps=10, hidden\_dim=256, epochs=100.

\textbf{Result}: Val=0.642, \textbf{Test=0.614}. Solid performance with the simplest possible meta-learning update.

%----------------------------------------------------------------------
\subsection{Reptile Variants}
%----------------------------------------------------------------------

\paragraph{Aggressive Reptile} (\texttt{numin\_reptile\_aggressive.py}): Larger model (hidden=512, 3-layer LSTM), SGD with momentum 0.9, 20 inner steps, GELU activations. \textbf{Result}: Test=\textbf{0.628}. Slightly better, suggesting more capacity and longer adaptation help.

\paragraph{Augmented Reptile} (\texttt{numin\_reptile\_augmented.py}): Adds task-level augmentation: Gaussian noise ($\sigma\!=\!0.02$), scale jitter ($\times U[0.8,1.2]$), time reversal (20\%), stock masking. \textbf{Result}: Test=\textbf{0.612}. \textbf{Negative result} --- augmentation slightly hurts performance. Financial returns have precise statistical properties that augmentation distorts.

%----------------------------------------------------------------------
\subsection{Prototypical Networks~\cite{snell2017}}
%----------------------------------------------------------------------

\textbf{Family}: Metric-based (Lectures~2, 8). \textbf{File}: \texttt{numin\_protonet.py}.

\textbf{Key idea}: Learn an embedding space where classification is performed by computing distances to class prototypes (Lecture~2, Master Equation).

\textbf{Adaptation for ranking}: Standard ProtoNet classifies into discrete classes. For 50-way ranking, we build \textbf{per-rank prototypes} by projecting $[\text{support\_encoding}, \text{rank\_embedding}]$ and accumulating into rank bins. The query prediction concatenates query encoding with each prototype and passes through an MLP decoder.

\textbf{No inner-loop adaptation} --- trained end-to-end with Adam (lr=0.001).

\textbf{Hyperparameters}: hidden\_dim=256, epochs=100.

\textbf{Result}: Val=0.653, \textbf{Test=0.639}. Strong performance without any inner-loop adaptation, suggesting the learned embedding space captures meaningful market structure.

%----------------------------------------------------------------------
\subsection{Conditional Neural Process (CNP)~\cite{garnelo2018}}
%----------------------------------------------------------------------

\textbf{Family}: Model-based (Lecture~8). \textbf{File}: \texttt{numin\_cnp.py}.

\textbf{Algorithm}: No inner loop. The model directly maps $F(x, \mathcal{D}_{\text{support}}) \to y$ in a single forward pass:
\begin{equation}
  r = \frac{1}{K}\sum_{i=1}^{K} g_\phi(x_i, y_i), \qquad \hat{y} = f_D\bigl([g(x_{\text{query}}); r]\bigr)
\end{equation}

\textbf{Architecture}: Encodes each support pair (window + rank targets) via LSTM + learned rank embeddings, aggregates via mean pooling to a task representation $r$, then decodes $[r; \text{query\_enc}] \to$ rank logits.

\textbf{Hyperparameters}: hidden\_dim=256, lr=0.001, epochs=100.

\textbf{Result}: Val=0.646, \textbf{Test=0.634}. The CNP fills the \textbf{model-based} gap in our taxonomy. Its strong performance shows that learning to \emph{read} the support context is nearly as effective as gradient-based adaptation for financial data.

%----------------------------------------------------------------------
\subsection{Stock Cross-Attention MAML}
%----------------------------------------------------------------------

\textbf{Family}: Model-based / Optimization-based hybrid. \textbf{File}: \texttt{numin\_attention.py}.

\textbf{Architecture}: Models stock-to-stock relationships explicitly. Each stock's temporal pattern is encoded independently via a per-stock LSTM, then cross-stock multi-head attention captures inter-stock correlations:
\begin{equation}
  Z_{\text{stock}} = \text{MultiHeadAttn}\bigl(\text{LSTM}(x_1), \ldots, \text{LSTM}(x_{50})\bigr) + \text{StockEmb}
\end{equation}

\textbf{Training}: MAML with \texttt{functional\_call} and \texttt{create\_graph=True}.

\textbf{Hyperparameters}: inner\_lr=0.01, outer\_lr=0.0005, inner\_steps=3, hidden\_dim=128, epochs=50.

\textbf{Result}: Val=0.649, \textbf{Test=0.643}. Strong result --- the explicit modeling of cross-stock relationships, combined with MAML adaptation, produces the second-best test correlation.

%----------------------------------------------------------------------
\subsection{Transformer MAML}
%----------------------------------------------------------------------

\textbf{Family}: Optimization-based. \textbf{File}: \texttt{numin\_transformer.py}.

\textbf{Architecture}: Replaces the LSTM encoder with a full Transformer:
\begin{equation}
  \text{TransformerEnc}: x \xrightarrow{\text{Linear}} \mathbb{R}^{W \times H} \xrightarrow{+\text{PosEnc}} \xrightarrow{\text{TransformerLayer} \times L} \xrightarrow{\text{LN+Pool}} \mathbb{R}^{H}
\end{equation}
Learnable positional encodings, 4 attention heads, 3 transformer layers.

\textbf{Training}: MAML with \texttt{functional\_call}.

\textbf{Result}: Val=0.648, \textbf{Test=0.622}. Moderate performance --- the Transformer's self-attention provides a different inductive bias from the LSTM's sequential processing, but doesn't consistently outperform it on this relatively small dataset.

%----------------------------------------------------------------------
\subsection{Ensemble Methods}
%----------------------------------------------------------------------

\paragraph{LSTM+Transformer Ensemble} (\texttt{numin\_ensemble.py}): Combines an LSTM model and a Transformer model with \textbf{learned softmax weights}: $\hat{y} = w_1 \cdot f_{\text{LSTM}}(x) + w_2 \cdot f_{\text{Trans}}(x)$. Trained with Reptile. \textbf{Result}: Test=\textbf{0.623}.

\paragraph{Multi-Seed Ensemble} (\texttt{numin\_ensemble\_seeds.py}): Averages softmax predictions from multiple Reptile models trained with different seeds. \textbf{Result}: Depends on available checkpoints; demonstrates that diversity through random initialization can improve robustness.

\section{Analysis \& Key Findings}
%==============================================================================

\subsection{Finding 1: ANIL is the Best Method ($\rho_s = 0.670$)}
ANIL~\cite{raghu2020} outperforms all other methods by a significant margin. This confirms the \textbf{feature reuse hypothesis}: the LSTM body learns general temporal features that transfer across market regimes, and only the linear ranking head needs task-specific adaptation. The reduced inner-loop computation (only updating $50 \times 2500 = 125{,}000$ head parameters vs.\ $\sim$1.3M total) also reduces gradient noise.

\subsection{Finding 2: All Methods Now Work ($\rho_s > 0.56$)}
After fixing critical implementation bugs:
\begin{itemize}[nosep]
  \item \textbf{MAML gradient flow}: Replaced \texttt{deepcopy}+SGD (which severed the computation graph) with \texttt{functional\_call} + \texttt{create\_graph=True}
  \item \textbf{ANIL body update}: Replaced Reptile-style interpolation (which gave zero body gradients) with proper meta-gradients
  \item \textbf{Data leakage}: Fixed normalization to use support-only statistics; fixed temporal split (chronological, no shuffling)
  \item \textbf{ProtoNet prototypes}: Fixed broken construction where all prototypes converged to the same value
\end{itemize}
Previously, MAML achieved $-0.01$ and Attention MAML achieved $0.068$. After fixes, they achieve $0.565$ and $0.643$ respectively.

\subsection{Finding 3: Optimization-based Methods Dominate}
The top 3 methods (ANIL, Attention MAML, FOMAML) are all optimization-based. Gradient-based task adaptation is crucial for financial data because market regimes shift between months --- the model must \emph{actively adapt} to new conditions, not just retrieve from a fixed embedding space.

\subsection{Finding 4: Model-based Methods are Competitive}
CNP (0.634) and Attention MAML (0.643) show that model-based approaches can be competitive. The CNP's ability to \emph{read} support context in a single forward pass is nearly as effective as gradient-based adaptation, suggesting that financial task structure can be captured through conditional computation.

\subsection{Finding 5: Task Augmentation Hurts Financial Data}
Augmented Reptile (0.612) $<$ vanilla Reptile (0.614). Financial returns have precise statistical properties; noise injection, scaling, and time reversal distort the signal. This contrasts with computer vision, where augmentation is always beneficial.

\subsection{Finding 6: Second-Order Gradients are Counterproductive}
Full MAML (0.565) $<$ FOMAML (0.631) $<$ ANIL (0.670). The second-order gradient computation amplifies noise in financial data. First-order approximations are both faster and more accurate for this domain.

\section{Conclusion}
%==============================================================================

We applied \textbf{11 meta-learning methods} from all three families (optimization-based, metric-based, model-based) to the Numin2 stock rank prediction benchmark. Our key contributions:

\begin{enumerate}[nosep]
  \item \textbf{ANIL achieves state-of-the-art} ($\rho_s = 0.670$), confirming the feature reuse hypothesis for financial data
  \item \textbf{All methods produce meaningful results} after fixing critical implementation bugs in MAML gradient flow, ANIL body updates, and data handling
  \item \textbf{Chronological splitting and support-only normalization} are essential for financial meta-learning to avoid temporal leakage
  \item \textbf{First-order methods outperform second-order}: FOMAML (0.631) $>$ MAML (0.565), and ANIL's head-only adaptation (0.670) beats full parameter adaptation
  \item \textbf{Model-based methods are competitive}: CNP (0.634) shows that learning to read context can match gradient-based adaptation
\end{enumerate}
