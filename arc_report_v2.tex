\section{Introduction \& Problem Setup}
%==============================================================================

\subsection{The 1D-ARC Benchmark}
The \textbf{1D Abstraction and Reasoning Corpus} (1D-ARC)~\cite{arc1d} is a simplified variant of Chollet's ARC benchmark~\cite{chollet2019}, designed to test whether a model can infer transformation rules from a small number of input--output examples and generalize to an unseen input. Unlike standard few-shot classification where the label space is shared across tasks, each ARC task defines a \emph{unique} transformation rule that must be discovered from scratch.

\begin{tcolorbox}[colback=blue!3,colframe=blue!40!black,title=\textbf{Task Definition}]
\begin{itemize}[nosep,leftmargin=*]
  \item \textbf{Dataset}: 901 tasks across 18 transformation types (move, copy, fill, flip, mirror, denoise, recolor, scale, etc.)
  \item \textbf{Each task}: 3 training (input $\to$ output) pairs + 1 test input
  \item \textbf{Input/Output}: 1D arrays of colored tokens (values 0--9), variable length ($\leq 100$)
  \item \textbf{Padding}: Dedicated token index 10 (value 0 is a valid color --- black)
  \item \textbf{Vocabulary}: 11 tokens (0--9 = colors, 10 = padding); output classes: 10
  \item \textbf{Metrics}: \textbf{Exact Match (EM)} --- 1 if the entire predicted sequence matches ground truth on every non-padding token, 0 otherwise. \textbf{Token Accuracy} --- fraction of individual non-padding tokens correctly predicted.
  \item \textbf{Split}: 70/15/15 train/val/test (630/135/136 tasks)
\end{itemize}
\end{tcolorbox}

\subsection{Task Types (18 Categories)}
The tasks span a diverse range of transformations requiring different levels of structural understanding:

\toprule
\textbf{Category} & \textbf{Tasks} & \textbf{Category} & \textbf{Tasks} \\

\end{center}

\subsection{Why Meta-Learning for ARC?}
ARC is the \textbf{canonical few-shot reasoning benchmark} (Lecture~11). Each task is a standalone learning problem with only 3 training examples --- exactly the meta-learning setup described in Lecture~5. Lecture~12 explicitly covers \emph{MAML Applied to ARC}. For context, on the full 2D ARC benchmark: GPT-4 achieves 15\%, o1 achieves 20\%, o3 achieves 85\%, and HRM achieves 40\% (Lecture~11).

\subsection{Formal Formulation (Lecture~5)}
\begin{equation}
  T_k = \bigl\{\mathcal{D}_{\text{train}}^k, \mathcal{D}_{\text{test}}^k\bigr\}, \quad \mathcal{D}_{\text{train}}^k = \bigl\{(\text{input}_i, \text{output}_i)\bigr\}_{i=1}^3, \quad \mathcal{D}_{\text{test}}^k = \bigl\{(\text{input}_{\text{test}}, ?)\bigr\}
\end{equation}
This is a \textbf{3-shot} learning problem: the model must infer a transformation rule from 3 examples and apply it to a novel input. The meta-learner trains across hundreds of tasks so that it can rapidly adapt to any new transformation type at test time.

The Hebbian layer adapts its weights on support examples \emph{before} predicting the query, providing a complementary adaptation mechanism to gradient-based meta-learning. \section{Shared Architecture}
%==============================================================================

All methods (except ProtoNet, which uses a Transformer encoder) share a common CNN-based architecture. The meta-learning algorithm determines \emph{how} the model adapts to each new task.

\subsection{Sequence Encoder: 3-Layer 1D CNN (Lectures~4, 5)}
The encoder maps variable-length token sequences to per-position feature vectors:
\begin{equation}
  x \in \{0,\ldots,10\}^L \xrightarrow{\text{Embed}(11 \to 64)} \mathbb{R}^{L\times64} \xrightarrow{\text{Conv1D}+\text{ReLU}+\text{LN}} \xrightarrow{\times 3} \mathbb{R}^{L\times128}
\end{equation}
\begin{itemize}[nosep]
  \item \textbf{Embedding}: \texttt{nn.Embedding(11, 64, padding\_idx=10)} --- a dedicated padding token (index 10) ensures that the embedding for color 0 (black) is learnable. Previous implementations incorrectly used \texttt{padding\_idx=0}, which zeroed out gradients for the most common color value.
  \item \textbf{Conv layers}: 3 blocks of Conv1D (kernel size 3, padding 1) $\to$ ReLU $\to$ LayerNorm with residual connections. Kernel size 3 preserves sequence length while capturing local patterns --- Lecture~4: \emph{``CNNs are excellent for local patterns, shift-invariant, and computationally efficient.''}
\end{itemize}

\subsection{Cross-Attention to Support Examples (Lectures~2, 5, 6)}
The query input's encoded features cross-attend to the aggregated support set representation using the \textbf{Master Equation} from Lecture~2:
\begin{equation}
  \hat{Y} = \text{softmax}\!\left(\frac{f(\hat{X}) \cdot g(X_{\text{support}})^\top}{\sqrt{d_k}}\right) \cdot h(Y_{\text{support}})
\end{equation}
Support examples are encoded as (input, output) pairs: each pair's input encoding and output encoding are concatenated along the sequence dimension and then averaged across the $K=3$ support examples to form a single task representation vector. The query's per-position features then cross-attend to this representation via multi-head attention (4 heads), allowing each query position to retrieve relevant information from the support context.

This is an instance of Matching Networks~\cite{vinyals2016} (Lecture~6): \emph{``If the encoder sees the support set, it knows which discriminative features to extract.''}

\subsection{Decoder: Per-Position Classification}
A 2-layer MLP maps each position's attended features to class logits:
\begin{equation}
  \text{Decoder}: \mathbb{R}^{L \times 128} \xrightarrow{\text{Linear}(128 \to 128)+\text{ReLU}} \xrightarrow{\text{Linear}(128 \to 10)} \mathbb{R}^{L \times 10}
\end{equation}
Each position independently predicts one of 10 color classes (0--9). The loss is cross-entropy with \texttt{ignore\_index=10}, which excludes padding positions from the gradient computation.

\section{Methods Implemented}
%==============================================================================

We implemented \textbf{7 methods} spanning all three meta-learning families from Lecture~10's taxonomy. Below we describe each method, its theoretical foundation, implementation details, and the rationale for including it.

%----------------------------------------------------------------------
\subsection{MAML + Hebbian TTA~\cite{finn2017} \textnormal{\textcolor{novelorange}{(Novel Combination)}}}
%----------------------------------------------------------------------
\textbf{Family}: Optimization-based (Lecture~6). \textbf{File}: \texttt{arc\_1d\_maml.py}.

\subsubsection{Algorithm}
MAML (Model-Agnostic Meta-Learning) learns an initialization $\theta$ such that a small number of gradient steps on a new task's support set produces a model that performs well on that task's query:
\begin{align}
  \textbf{Inner loop (adapt):} \quad & \theta'_k = \theta - \alpha \nabla_\theta \mathcal{L}\bigl(\mathcal{D}_{\text{support}}^k; \theta\bigr) \quad \text{(repeated for $S$ steps)} \\
  \textbf{Outer loop (meta-update):} \quad & \theta \leftarrow \theta - \beta \nabla_\theta \mathcal{L}\bigl(\mathcal{D}_{\text{query}}^k; \theta'_k\bigr)
\end{align}
The key insight (Lecture~6) is that the outer loop differentiates \emph{through} the inner loop, requiring second-order gradients ($\nabla^2_\theta \mathcal{L}$). This is computationally expensive but finds initializations that are maximally sensitive to task-relevant gradient directions.

\subsubsection{Implementation}
We use \texttt{torch.func.functional\_call} to perform differentiable forward passes with cloned parameter dictionaries. Inner-loop gradients use \texttt{create\_graph=True} to preserve the computation graph for the outer loop. Efficient SDPA is disabled (\texttt{enable\_flash\_sdp(False)}) because the scaled dot-product attention kernel does not support double backward.

To prevent \textbf{information leakage}, the inner loop uses a \textbf{leave-one-out} strategy: when adapting on support example $j$, that example is excluded from the context, so the model cannot trivially copy the answer from its own input.

\subsubsection{Novel: Hebbian Test-Time Adaptation}
In addition to gradient-based MAML, we add a \textbf{Hebbian learning layer} that adapts its weights online during inference using Oja's rule~\cite{oja1982}:
\begin{equation}
  \Delta W = \eta\bigl(yx^\top - \text{diag}(y^2) \cdot W\bigr)
\end{equation}
The Hebbian layer is inserted between the cross-attention output and the decoder. During forward passes with \texttt{adapt=True}, the layer updates its weights from support examples \emph{before} processing the query. This provides a complementary, biologically-inspired adaptation mechanism: while MAML adapts through backpropagation, the Hebbian layer adapts through local, correlation-based learning.

\textbf{Config}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=5, batch\_size=4, epochs=100.

%----------------------------------------------------------------------
\subsection{FOMAML --- First-Order MAML~\cite{finn2017}}
%----------------------------------------------------------------------
\textbf{Family}: Optimization-based (Lecture~6). \textbf{File}: \texttt{arc\_fomaml.py}.

\subsubsection{Algorithm}
FOMAML approximates MAML by dropping the second-order gradient terms. Instead of differentiating through the inner loop, it treats the adapted parameters $\theta'_k$ as if they were independent of $\theta$:
\begin{equation}
  \theta \leftarrow \theta - \beta \nabla_{\theta'_k} \mathcal{L}\bigl(\mathcal{D}_{\text{query}}^k; \theta'_k\bigr) \quad \text{(stop-gradient through inner loop)}
\end{equation}
Lecture~8 shows that this first-order approximation retains most of MAML's performance while being significantly cheaper (no Hessian computation). The intuition is that the direction of the query-loss gradient at $\theta'_k$ is a good proxy for the meta-gradient at $\theta$.

\subsubsection{Implementation}
Uses \texttt{deepcopy(model)} to create a task-specific copy. After inner-loop adaptation via SGD (5 steps), the first-order gradients from the query loss on the adapted model are manually copied back to the meta-model's \texttt{.grad} tensors, and the Adam outer optimizer steps on those. Leave-one-out context is used in the inner loop.

\textbf{Config}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=5, epochs=100.

%----------------------------------------------------------------------
\subsection{ANIL --- Almost No Inner Loop~\cite{raghu2020}}
%----------------------------------------------------------------------
\textbf{Family}: Optimization-based. \textbf{File}: \texttt{arc\_anil.py}.

\subsubsection{Algorithm}
Raghu et al.~\cite{raghu2020} showed that MAML's effectiveness comes primarily from \textbf{feature reuse}, not rapid learning: the encoder body learns general-purpose features during meta-training, and only the final classification head needs task-specific adaptation. ANIL formalizes this:
\begin{align}
  \textbf{Inner loop:} \quad & \text{head}'_k = \text{head} - \alpha \nabla_{\text{head}} \mathcal{L}_{\text{support}}^k \quad \text{(body \textbf{frozen})} \\
  \textbf{Outer loop:} \quad & [\text{body}; \text{head}] \leftarrow [\text{body}; \text{head}] - \beta \nabla_{[\text{body};\text{head}]} \mathcal{L}_{\text{query}}^k(\theta'_k) \quad \text{(\textbf{all} updated)}
\end{align}
The inner loop only adapts the decoder head (a small linear layer), dramatically reducing inner-loop computation. The outer loop still updates all parameters through meta-gradients, so the body continues to improve its feature representations over the course of meta-training.

\subsubsection{Implementation}
Uses \texttt{functional\_call} with \texttt{create\_graph=True}. Parameter keys containing \texttt{``head''} or \texttt{``dec''} are identified as head parameters; only these receive gradient updates in the inner loop. The query loss backpropagates through the full computation graph to update body + head in the outer loop.

\textbf{Config}: inner\_lr=0.01, outer\_lr=0.001, inner\_steps=5, epochs=100.

%----------------------------------------------------------------------
\subsection{Reptile~\cite{nichol2018}}
%----------------------------------------------------------------------
\textbf{Family}: Optimization-based (Lecture~8). \textbf{Files}: \texttt{arc\_reptile.py}, \texttt{arc\_reptile\_is20.py}.

\subsubsection{Algorithm}
Reptile is the simplest optimization-based meta-learner. It runs multiple steps of SGD on a task's support set, then moves the initialization toward the adapted weights:
\begin{equation}
  \boxed{\theta \leftarrow \theta + \epsilon\bigl(\tilde{\theta} - \theta\bigr)} \quad \text{where } \tilde{\theta} = \text{SGD}^K(\theta, \mathcal{D}_{\text{support}})
\end{equation}
Lecture~8: \emph{``Drop all second-order terms. Move the initialization $\theta$ toward the adapted parameters $\tilde{\theta}$.''} The geometric intuition is that $\theta$ moves toward the \textbf{centroid} of all task-specific optima, finding a point equidistant from all tasks in parameter space.

Nichol et al.~\cite{nichol2018} showed that Reptile implicitly approximates the full MAML gradient: the difference is only in higher-order Taylor expansion terms.

\subsubsection{Implementation}
We use linear $\epsilon$ decay following the original paper: $\epsilon_t = \epsilon_0 \cdot (1 - t/T)$, annealing from the initial step size to zero over training. Leave-one-out context prevents leakage. We tested two variants: 10 inner steps (standard) and 20 inner steps (extended adaptation).

\textbf{Config}: inner\_lr=0.01, outer\_lr=0.1, inner\_steps=10 or 20, epochs=100/150.

%----------------------------------------------------------------------
\subsection{Prototypical Networks~\cite{snell2017}}
%----------------------------------------------------------------------
\textbf{Family}: Metric-based (Lectures~2, 8). \textbf{File}: \texttt{arc\_protonet.py}.

\subsubsection{Algorithm}
ProtoNet learns an embedding space where classification is performed by computing distances to class prototypes. For each output color class $c \in \{0, \ldots, 9\}$, a prototype is computed as the mean of all per-position features from support outputs labeled with that class:
\begin{equation}
  p_c = \frac{1}{|\mathcal{S}_c|} \sum_{(i,j) \in \mathcal{S}_c} f_\phi(x_i)_j, \quad \text{where } \mathcal{S}_c = \{(i,j) : y_{i,j} = c, \, \text{mask}_{i,j} = 1\}
\end{equation}
Classification is then by negative squared Euclidean distance:
\begin{equation}
  P(y_j = c \mid x_{\text{query}}) \propto \exp\!\bigl(-\|f_\phi(x_{\text{query}})_j - p_c\|^2\bigr)
\end{equation}

\subsubsection{Architecture}
Unlike other methods, ProtoNet uses a \textbf{Transformer-based encoder} (3 layers, 4 heads, hidden dim 256) instead of CNN, since the distance-based prediction requires richer per-position representations. Support masks filter out padding positions from prototype computation.

\textbf{No inner-loop adaptation} --- trained end-to-end with Adam. The embedding space must be learned to be discriminative without any test-time gradient updates.

\textbf{Config}: lr=0.001, hidden\_dim=256, embed\_dim=128, epochs=100.

%----------------------------------------------------------------------
\subsection{Conditional Neural Process (CNP)~\cite{garnelo2018}}
%----------------------------------------------------------------------
\textbf{Family}: Model-based (Lecture~8). \textbf{File}: \texttt{arc\_cnp.py}.

\subsubsection{Algorithm}
The CNP is a \textbf{model-based meta-learner} that performs task adaptation in a single forward pass without any gradient-based inner loop. It encodes each support (input, output) pair into a representation, aggregates all pairs via mean pooling into a single task representation $r$, and conditions the query prediction on $r$:
\begin{equation}
  r = \frac{1}{K}\sum_{i=1}^{K} g_\phi\bigl(\text{enc}(\text{input}_i) \oplus \text{enc}(\text{output}_i)\bigr), \qquad \hat{y}_j = f_D\bigl([\text{enc}(x_{\text{query}})_j \;;\; r]\bigr)
\end{equation}
The aggregation uses \textbf{mask-aware mean pooling}: padding positions are excluded by weighting each position by its mask value before averaging, preventing the task representation from being diluted by padding tokens.

This approach is related to \textbf{in-context learning} (Lecture~13): the model ``reasons'' about the transformation rule by reading the support examples, analogous to how large language models learn from in-context examples.

\textbf{Config}: lr=0.001, hidden\_dim=128, epochs=100.

%----------------------------------------------------------------------
\subsection{Matching Networks~\cite{vinyals2016}}
%----------------------------------------------------------------------
\textbf{Family}: Metric-based (Lecture~6). \textbf{File}: \texttt{arc\_matching.py}.

\subsubsection{Algorithm}
Matching Networks classify by attention-weighted comparison to support examples. The core idea from Lecture~6: \emph{``The test input is classified by comparing it to the labeled support set using a learned similarity function.''}

Our implementation uses \textbf{per-position attention} rather than the standard mean-pooled approach, which is critical for ARC's position-dependent transformations:
\begin{enumerate}[nosep]
  \item \textbf{Keys}: All per-position CNN-encoded features from support inputs (filtered by mask to exclude padding). If there are $K=3$ support examples each of length $L$, we get up to $3L$ key vectors.
  \item \textbf{Values}: Per-position embeddings of support output tokens (also filtered by mask). Each value carries the information ``what color should this position have.''
  \item \textbf{Query}: Per-position CNN-encoded features from the test input.
  \item \textbf{Attention}: Scaled dot-product attention from each query position to all support positions:
  \begin{equation}
    \text{attended}_j = \sum_{i} \frac{\exp(q_j \cdot k_i / \sqrt{d})}{\sum_{i'} \exp(q_j \cdot k_{i'} / \sqrt{d})} \cdot v_i
  \end{equation}
  \item \textbf{Decoder}: The attended values are concatenated with the query's own features: $[\text{query\_features}_j \;;\; \text{attended}_j] \to$ MLP $\to$ 10-class logits.
\end{enumerate}

This design preserves \textbf{positional correspondence} --- each query position can attend to the most relevant support positions (e.g., ``position 5 in the input corresponds to position 7 in the output''), which is essential for spatial transformation tasks. No inner loop is needed.

\textbf{Config}: lr=0.001, hidden\_dim=128, epochs=100.

\section{Key Findings}
%==============================================================================

\subsection{Finding 1: Matching Networks Achieves 61.8\% EM}
Matching Networks achieves the highest Exact Match by a significant margin (+5.2\% over FOMAML). The critical factor is \textbf{per-position attention}: by attending to individual support positions rather than aggregated representations, the model can directly discover positional correspondences between inputs and outputs. For transformations like ``move 2 pixels right,'' the attention mechanism learns to shift its focus by exactly 2 positions.

\subsection{Finding 2: FOMAML and ANIL Achieve Strong EM (56.6\%, 55.9\%)}
Both first-order optimization methods produce strong results. Notably, ANIL achieves the highest \emph{token} accuracy (93.9\%) but lower EM than FOMAML (55.9\% vs.\ 56.6\%), suggesting that ANIL makes fewer errors per sequence on average but those errors are more spread across sequences. FOMAML, despite lower token accuracy, produces more ``all-or-nothing'' predictions.

\subsection{Finding 3: CNP Solves 52.9\% of Tasks Without Any Gradient Updates}
The CNP achieves over half of tasks exactly correct using \emph{only a single forward pass} --- no inner-loop gradient computation at all. This is remarkable and connects to Lecture~13's in-context learning: the model has learned to ``read'' the support examples and extract the transformation rule through pure conditional computation.

\subsection{Finding 4: Reptile Shows High Token Accuracy but Low EM}
Reptile achieves 87.8\% token accuracy but only 38.2\% EM. The interpolation-based meta-update ($\theta \leftarrow \theta + \epsilon(\tilde{\theta} - \theta)$) produces good \emph{average} predictions but lacks the precision required for exact sequence reproduction. Its errors are distributed across many sequences --- it gets most tokens right but rarely gets \emph{all} of them right.

\subsection{Finding 5: ProtoNet Gets 0\% EM}
Distance-based classification without task-specific adaptation is fundamentally insufficient for ARC. The same color token (e.g., ``3'') can serve different \emph{functional roles} across tasks --- as a target, a boundary marker, or a fill value. A fixed embedding space cannot capture these task-dependent semantics, and ProtoNet never produces a single fully correct sequence.

\subsection{Finding 6: Local vs.\ Global Task Difficulty}
The per-task analysis reveals that CNN-based models achieve near-perfect EM on local operations (move: 100\%, hollow: 98\%, denoise: 94\%) but struggle with global operations (mirror: 18\%, padded fill: 20\%). This gap suggests that a Transformer-based encoder (which can attend globally) would significantly improve performance on the hardest task types.

\section{Connection to Reasoning (Lectures~11--13)}
%==============================================================================

Lecture~11 introduces the \textbf{Hypothesis-Refinement Model (HRM)} for ARC, which iteratively refines its understanding of a task through multiple rounds of processing. Our Reptile inner loop is a simplified analogue: each SGD step refines the model's ``hypothesis'' about the current task's transformation rule. Lecture~11: \emph{``Chain-of-thought transforms bounded-depth computation into unbounded-depth computation.''} Similarly, more inner steps allow deeper ``reasoning'' ($\text{is20} = 35.3\%$ EM vs.\ $\text{is10} = 38.2\%$ EM --- though diminishing returns suggest that the CNN encoder's limited receptive field is the bottleneck, not the number of adaptation steps).

Lecture~12 frames ARC explicitly as meta-learning: \emph{``Each ARC task has the standard meta-learning structure.''} Our results confirm this framing and show that even simple meta-learners can achieve meaningful performance (61.8\% EM) on abstract reasoning.

The success of the CNP (52.9\% EM without any inner loop) suggests an intriguing connection to \textbf{in-context learning} (Lecture~13): the CNP processes all support examples in a single forward pass, learning to extract and apply transformation rules purely through conditional computation --- analogous to how large language models ``learn'' from in-context examples without weight updates.

%==============================================================================
\section{Conclusion}
%==============================================================================

We evaluated \textbf{7 meta-learning methods} from all three families on the 1D-ARC abstract reasoning benchmark. Our main findings:
\begin{enumerate}[nosep]
  \item \textbf{Matching Networks} achieves the best Exact Match of \textbf{61.8\%} (token accuracy 92.1\%), enabled by per-position attention that preserves spatial correspondences
  \item \textbf{FOMAML} (EM=56.6\%) and \textbf{ANIL} (EM=55.9\%) demonstrate that first-order optimization-based methods are competitive, with ANIL confirming the feature reuse hypothesis
  \item \textbf{CNP} achieves 52.9\% EM \emph{without any inner loop}, showing that model-based in-context reasoning is a viable approach to abstract reasoning
  \item A \textbf{novel Hebbian TTA + MAML combination} provides a biologically-inspired adaptation mechanism beyond standard gradient-based meta-learning
  \item Per-task analysis reveals a clear \textbf{local/global dichotomy}: CNN encoders excel at local transformations (100\% EM on move tasks) but struggle with global ones (18\% EM on mirror)
  \item \textbf{ProtoNet achieves 0\% EM}, confirming that fixed-embedding metric learning without task adaptation is insufficient for abstract reasoning
\end{enumerate}
