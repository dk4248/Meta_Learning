Numin2: Meta-Learning for Stock Rank Prediction

11 methods • 50 Nifty-50 stocks • Monthly tasks • Spearman correlation

Problem

Predict relative rankings of 50 Nifty-50 Indian stocks by next-day returns. Each month is a separate meta-learning task with different market dynamics.

Input50-day window of returns for 50 stocks (50×50 matrix)
OutputRank ordering of 50 stocks (0-49)
SupportFirst 5 days of the month
QueryRemaining days
MetricSpearman rank correlation ρs
SplitChronological 70/15/15 (no temporal leakage)

ANIL — Almost No Inner Loop Optimization — ρ=0.670

Best method. Only the decoder head adapts in the inner loop; the BiLSTM+Attention body is frozen during adaptation but updated via meta-gradients in the outer loop. Confirms the feature reuse hypothesis: the encoder learns market-regime-agnostic temporal features, and only the ranking head needs monthly adaptation.

Inner: head' = head - α ∇_head L_support [body frozen, 3 steps] Outer: [body; head] ← [body; head] - β ∇ L_query(θ') [all updated]

Uses functional_call + create_graph=True. Hidden=256, Adam outer optimizer.

Stock Cross-Attention MAML OptimizationNovel — ρ=0.643

Novel architecture. Each stock's temporal pattern is encoded independently via a per-stock LSTM, then cross-stock multi-head attention captures inter-asset correlations. Learned stock embeddings provide identity information.

Z_stock = MultiHeadAttn(LSTM(x_1), ..., LSTM(x_50)) + StockEmb

Trained with MAML (functional_call). The explicit modeling of cross-stock relationships provides the second-best correlation.

MAML Optimization — ρ=0.565

Full second-order MAML with BiLSTM+Attention encoder. cuDNN disabled for LSTM double backward compatibility. Second-order gradients amplify noise in financial data, resulting in the lowest correlation among optimization methods.

FOMAML Optimization — ρ=0.631

First-order approximation. Uses deepcopy + SGD, copies gradients back. Significantly better than full MAML (0.631 vs 0.565) — dropping second-order gradients actually helps on noisy financial data.

Reptile Optimization — ρ=0.614

Simplest meta-learner: SGD on support, interpolate toward adapted weights. Linear ε decay. Three variants tested:

Standard (hidden=256, is=10)ρ=0.614
Aggressive (hidden=512, is=20, momentum)ρ=0.628
Augmented (noise, scale, reversal, masking)ρ=0.612

Negative result: Task augmentation slightly hurts financial meta-learning — financial returns have precise statistical properties that augmentation distorts.

Prototypical Networks Metric-based — ρ=0.639

Builds per-rank prototypes from support encodings + learned rank embeddings. Query prediction via prototype-conditioned MLP decoder. No inner loop. Strong performance (3rd overall) suggests the embedding space captures meaningful market structure.

Conditional Neural Process Model-based — ρ=0.634

No inner loop. Encodes support (window, rank) pairs via LSTM + rank embeddings, aggregates to task representation via mean pooling, decodes query conditioned on task representation. Single forward pass adaptation.

Ensemble Methods Hybrid

LSTM+Transformer Ensemble (ρ=0.623): Combines LSTM and Transformer models with learned softmax weights. Trained with Reptile.

Multi-Seed Ensemble: Averages softmax predictions from Reptile models trained with different random seeds.

Key Findings

1. ANIL Dominates (ρ=0.670)

Head-only adaptation outperforms full-parameter methods. The BiLSTM body learns general temporal features that transfer across market regimes.

2. First-Order > Second-Order

FOMAML (0.631) > MAML (0.565). Second-order gradients amplify noise in financial data. Simpler is better.

3. All Methods Work (ρ > 0.56)

After fixing MAML gradient flow, ANIL body updates, and data handling, every method produces meaningful correlations.

4. Augmentation Hurts Financial Data

Augmented Reptile (0.612) < Reptile (0.614). Financial returns have precise statistical properties — noise/scaling/reversal distorts the signal.
