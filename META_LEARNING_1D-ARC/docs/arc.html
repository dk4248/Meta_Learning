MAML + Hebbian Test-Time Adaptation OptimizationNovel

Model-Agnostic Meta-Learning learns an initialization that adapts well in few gradient steps. We add a Hebbian layer (Oja's rule) that adapts online from support examples before predicting the query.

θ' = θ - α ∇θ L(D_support; θ)   [inner loop, repeated S times]
θ ← θ - β ∇θ L(D_query; θ')   [outer loop, differentiates through inner]

Hebbian: ΔW = η(yxᵀ - diag(y²)W)   [Oja's rule, adapts before query prediction]

Uses torch.func.functional_call with create_graph=True. Leave-one-out inner loop prevents information leakage.

FOMAML — First-Order MAML Optimization

Drops second-order gradients for efficiency. Uses deepcopy for inner loop, copies first-order gradients back to meta-model. Same accuracy as full MAML but much faster.

θ ← θ - β ∇θ' L(D_query; θ')   [stop gradient through inner loop]

Result: EM=56.6%, Token=91.0%. Best speed-accuracy tradeoff.

ANIL — Almost No Inner Loop Optimization

Only the decoder head adapts in the inner loop; the encoder body is frozen. Confirms the feature reuse hypothesis: the encoder learns general features, only the head needs task-specific adaptation.

Inner: head' = head - α ∇_head L_support   [body frozen]
Outer: [body; head] ← [body; head] - β ∇ L_query(θ')   [all updated via meta-gradients]

Result: EM=55.9%, Token=93.9% (highest token accuracy!).

Reptile Optimization

The simplest meta-learner. Run SGD on a task, then interpolate toward adapted weights. Geometric intuition: θ moves toward the centroid of all task optima.

θ ← θ + ε(θ̃ - θ)   where θ̃ = SGD^K(θ, D_support)

Linear ε decay: εt = ε0(1 - t/T). Two variants: 10 steps (EM=38.2%) and 20 steps (EM=35.3%).

Matching Networks Metric-based

Best method (EM=61.8%). Per-position attention: each query position attends to all support positions to retrieve output information.

attended_j = Σ_i softmax(q_j · k_i / √d) · v_i
logits = Decoder([query_features; attended_values])

Keys = encoded support inputs. Values = embedded support output tokens. No inner loop needed.

Conditional Neural Process Model-based

No inner loop. Encodes support pairs, aggregates to task representation, conditions query prediction on it. Related to in-context learning.

r = mean(g(input_i, output_i))   [task representation]
ŷ = Decoder([encode(x_query); r])

Result: EM=52.9%. Solves half of tasks without any gradient updates.

Prototypical Networks Metric-based

Builds per-class prototypes (one per color 0-9) from support outputs. Classifies by negative squared Euclidean distance. Uses Transformer encoder.

p_c = mean(f(x_i)_j) for positions where y=c
P(y_j=c) ∝ exp(-||f(x_query)_j - p_c||²)

Result: EM=0.0%. Fixed embeddings cannot capture task-dependent token semantics.
